# 区别使用文档

本文档说明当前项目相对远端 `zrwswzl/note-sharing:main` 的主要区别，以及同事拉取合并后的使用方式。

## 一、核心变化概览

当前改动主要集中在三块：AI 助手、问答区 AI 辅助、后端迁移和本地开发脚本。

### 1. AI 助手增强

- AI 面板支持流式回答。
- AI 回答支持 Markdown 渲染。
- AI 回答中的站内笔记/问答引用支持点击跳转。
- 支持站内内容检索，不再只依赖当前页面上下文。
- 无检索结果时返回统一提示，避免生成不存在的文章、笔记或问答。
- 统一 BFF 不可用、未登录、无权限、停止生成、起草失败等用户提示。

### 2. 问答区 AI 辅助

- 提问弹窗输入标题或正文时，会自动检测相似问题。
- 相似问题检测失败不会阻塞正常发帖。
- 新增 AI 起草问题能力，可根据关键词生成标题、正文和标签。
- AI 草稿需要用户手动采纳，不会自动发布。
- 全局 AI 面板在问答区也可以生成问题草稿，并带入发帖弹窗。

### 3. AI BFF 安全和稳定性

- AI BFF 使用 JWT 校验用户身份。
- 模型 API Key 只放在 `ai_bff/.env`，不暴露给前端。
- 输入长度增加限制，避免超长请求拖垮服务。
- 站内引用会校验真实存在后再返回。
- 对模型返回内容做站内引用后处理，避免伪造链接。

### 4. 后端和数据库变化

- 清理了不规范的 Flyway 迁移文件名。
- 新增规范迁移文件：
  - `V5__create_user_follow_table.sql`
  - `V7__create_note_stats.sql`
  - `V8__create_note_stats_compensation.sql`
  - `V10__create_user_favorite_note_table.sql`
- 新增站内笔记目录接口：
  - `GET /api/v1/search/notes/catalog`
- MinIO 默认本地配置改为 `minioadmin/minioadmin`。

### 5. 本地开发脚本

新增 `Starting/` 下的启动、停止和检查脚本，方便本地一次启动或关闭应用层服务：

- `Starting/start_apps.bat`
- `Starting/stop_apps.bat`
- `Starting/test/check_services.ps1`
- `Starting/test/run_gate.ps1`
- `Starting/test/run_bff_smoke.ps1`

## 二、同事拉取后的使用步骤

假设同事基础环境已经准备好：Java、Node.js、Python、MySQL、Redis、MongoDB、Elasticsearch、MinIO 等均可用。

### 1. 拉取代码

```powershell
git pull
```

### 2. 安装或更新前端依赖

本次前端新增了 `dompurify`。由于当前项目忽略了 `Login_ui/package-lock.json`，建议拉取后执行：

```powershell
cd Login_ui
npm install
```

然后可验证构建：

```powershell
npm.cmd run build
```

### 3. 配置 AI BFF

复制模板：

```powershell
cd ai_bff
copy .env.example .env
```

至少补齐：

```env
AI_MODEL_API_KEY=你的模型APIKey
AI_JWT_SECRET=与 Login_api/src/main/resources/application.yml 中 jwt.secret 一致
```

如果暂定全站只接一个模型，推荐保持：

```env
AI_MODEL_PROVIDER=openai_compatible
AI_MODEL_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_MODEL_NAME=qwen3-vl-plus
```

注意：真实 API Key 不要提交到 Git。

### 4. 启动服务

可以使用脚本：

```powershell
Starting\start_apps.bat
```

它会分别启动：

- 后端 API：`http://localhost:8080`
- 前端 UI：`http://localhost:8082`
- AI BFF：`http://localhost:8000`

也可以手动启动：

```powershell
cd Login_api
.\gradlew.bat bootRun
```

```powershell
cd Login_ui
npm run serve
```

```powershell
cd ai_bff
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 5. 数据库迁移注意事项

新环境或干净库通常可以直接启动，由 Flyway 自动迁移。

如果同事本地数据库曾经执行过旧的非规范迁移文件，例如：

- `V4_create_note_stat_table.sql`
- `V5_create_user_follow_table.sql`

可能需要根据本地 `flyway_schema_history` 情况执行 `flyway repair` 或手动处理历史记录。否则开启 `validate-migration-naming` 后，旧历史可能导致启动失败。

### 6. 快速验证

后端编译：

```powershell
cd Login_api
.\gradlew.bat compileJava
```

前端构建：

```powershell
cd Login_ui
npm.cmd run build
```

BFF 编译检查：

```powershell
cd ai_bff
python -m compileall app
```

## 三、能否直接使用

如果同事具备基础环境，并完成以下几项，本次改动可以直接使用：

- 执行 `npm install` 更新前端依赖。
- 配置 `ai_bff/.env` 中的 `AI_MODEL_API_KEY` 和 `AI_JWT_SECRET`。
- 数据库 Flyway 历史没有旧非法迁移冲突，或已修复。
- MinIO 本地账号与 `application.yml` 保持一致。

如果没有配置模型 API Key，普通后端和前端仍可运行，但 AI 模型调用能力不可用或只能走降级逻辑。

## 四、不应提交的本地文件

以下文件或目录属于本地运行、密钥或 SDD 工作区数据，不应提交：

- `API_KEY.txt`
- `ai_bff/.env`
- `.runtime/`
- `.cursor/`
- `specs/`
- `Login_ui/node_modules/`
- `Login_ui/dist/`
- `Login_api/build/`

当前 `.gitignore` 已覆盖这些主要风险项。
